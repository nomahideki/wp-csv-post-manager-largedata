プラグイン名：CSV投稿マネージャー（改造版）
動作内容：同時に多数の投稿をインポートする際に、サーバに負荷をかけず、実行できるようなファイル分割機能・バックグラウンド動作機能を追加しました。
作成者：Hideki Noma (Logitoy)
作成日：2017/01/17
更新日：2017/01/22

■■■　設置方法　■■■
１）csv-post-managerプラグインのインストール（インストールしてない場合）

２）ファイルのアップロード
WordPressのwp-content/pluginsディレクトリに
csv-post-manager-largedata
フォルダを設置

設置イメージ
wp-content/plugins/csv-post-manager-largedata
   |- csv-post-manager-largedata.php
   |- export.php
   |- work(フォルダ)
   |- readme.txt


３）プラグインの有効化
WordPress管理画面のプラグインマネジャーにて
・csv-post-manager(CSV投稿マネージャー)プラグインの無効化
・CSV Post Manager Large Dataの有効化
をしてください。

以上で設置は完了となります。

■■■　使用方法　■■■
CSVのインポート時にファイルが大きい場合、分割処理に対応しました。

１）「CSV投稿インポーター」を使用する際、「投稿をインポート（分割処理）」ボタンを使用してください。
２）上記ボタンを押すと、画面が切り替わり「処理中」となります。処理が終わるまで順次「・」が追記されていきますのでお待ちください。
３）処理が完了するとメッセージが「処理完了」に切り替わります。

※処理途中で画面を消してしまった場合は、「CSV投稿マネージャー」の画面を表示すると処理が再開しました。
　重複処理はしないようになってます。

■■■　注意事項　■■■
・CSV投稿マネージャーの機能はそのまま使用できるようになってますが、
　インポート時のエラーメッセージなどは確認できません。問題が起きた場合は小さいサイズのCSVファイルを通常のインポート処理する等の方法で動作確認をお願いいたします。
・CSV投稿マネージャーは、csv-post-managerというフォルダ名で設置してください。
　他の名称での動作は保証できません。
・CSV投稿マネージャーがバージョンアップした際も動作するように作成しておりますが、
　大幅な仕様変更が行われた場合には動作しなくなる場合がございます。バージョンアップ時にはご注意ください。

■■■　動作仕様　■■■
以下のような動作になってます。
・CSVファイルをアップロード
・プラグインのworkフォルダ内に、CSVファイルと設定情報を保存したファイルを一時的に作成します。
・JavaScriptでAjaxにより、CSVファイルがあるか確認
・CSVファイルがあれば、１ファイルずつ順番にインポート処理を実行

■■■　実装仕様　■■■
CSV投稿マネージャープラグインを再利用し、ボタンおよびバックグランドでのCSV処理手順を追加してます。
